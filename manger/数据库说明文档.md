# 在线考试系统数据库文档

## 数据库概述

数据库名称为 `user_auth_db`，基于 MySQL 8.1.0 构建，默认字符集为 `utf8mb4`，排序规则为 `utf8mb4_unicode_ci`，主要用于支撑在线考试系统的全流程运行，涵盖班级管理、课程管理、题库管理、考试管理、成绩管理等核心业务场景，通过多表关联实现数据的规范化存储与高效查询。

## 核心表结构详情

### 1. 班级课程关联表（class_courses）

用于记录班级与课程的关联关系，支持多对多映射。

| 字段名        | 类型        | 是否主键 | 默认值            | 注释                           |
| ------------- | ----------- | -------- | ----------------- | ------------------------------ |
| id            | bigint      | 是       | 自增              | 关联 ID                        |
| class_id      | bigint      | 否       | 无                | 班级 ID（关联 classes 表 id）  |
| course_id     | bigint      | 否       | 无                | 课程 ID（关联 courses 表 id）  |
| semester      | varchar(20) | 否       | NULL              | 开课学期                       |
| academic_year | varchar(10) | 否       | NULL              | 开课学年                       |
| is_active     | tinyint(1)  | 否       | 1                 | 是否有效（1 = 有效，0 = 无效） |
| created_at    | timestamp   | 否       | CURRENT_TIMESTAMP | 创建时间                       |

- 唯一索引：`uk_class_course`（class_id + course_id），确保班级与课程关联唯一
- 普通索引：`idx_class_id`（class_id）、`idx_course_id`（course_id）
- 外键约束：关联 classes 表 id、courses 表 id

### 2. 班级表（classes）

存储班级基础信息，关联专业和老师。

| 字段名           | 类型         | 是否主键 | 默认值                                        | 注释                           |
| ---------------- | ------------ | -------- | --------------------------------------------- | ------------------------------ |
| id               | bigint       | 是       | 自增                                          | 班级 ID                        |
| class_code       | varchar(50)  | 否       | 无                                            | 班级代码（唯一）               |
| class_name       | varchar(100) | 否       | 无                                            | 班级名称                       |
| grade            | varchar(10)  | 否       | 无                                            | 年级（如：2022）               |
| major_id         | bigint       | 否       | 无                                            | 专业 ID（关联 majors 表 id）   |
| class_number     | int          | 否       | 无                                            | 班级序号（如：1 班、2 班）     |
| teacher_id       | bigint       | 否       | 无                                            | 老师ID（关联 users 表 id）     |
| description      | text         | 否       | NULL                                          | 班级描述                       |
| max_students     | int          | 否       | 50                                            | 最大学生数                     |
| current_students | int          | 否       | 0                                             | 当前学生数                     |
| is_active        | tinyint(1)   | 否       | 1                                             | 是否启用（1 = 启用，0 = 禁用） |
| created_at       | timestamp    | 否       | CURRENT_TIMESTAMP                             | 创建时间                       |
| updated_at       | timestamp    | 否       | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间                       |

- 唯一索引：`uk_class_code`（class_code）、`uk_grade_major_class`（grade + major_id + class_number）
- 普通索引：`idx_major_id`（major_id）、`idx_teacher_id`（teacher_id）、`idx_grade`（grade）
- 外键约束：关联 majors 表 id、users 表 id

## 表关系图（文字描述）

1. 多对多关系：
   - 班级与课程：通过 class_courses 表关联
   - 题目与课程：通过 question_courses 表关联
   - 题目与知识点：通过 question_knowledge_points 表关联
   - 角色与菜单：通过 role_menus 表关联
   - 角色与权限：通过 role_permissions 表关联
2. 一对多关系：
   - 专业（majors）→ 班级（classes）：一个专业包含多个班级
   - 班级（classes）→ 考试（exams）：一个班级可参与多个考试
   - 课程（courses）→ 试卷（papers）：一个课程可对应多个试卷
   - 试卷（papers）→ 考试（exams）：一个试卷可用于多个考试
   - 角色（roles）→ 用户（users）：一个角色可分配给多个用户
3. 一对一关系：
   - 考试（exams）→ 试卷（papers）：一个考试对应一个试卷
   - 学生考试记录（student_exams）→ 学生答题记录（student_answers）：一条考试记录对应一条答题记录

## 数据库初始化说明

1. 数据库创建：默认字符集`utf8mb4`，排序规则`utf8mb4_unicode_ci`，禁用加密（ENCRYPTION='N'）
2. 表创建顺序：需先创建基础表（如 users、majors、roles、permissions），再创建关联表（如 classes、courses、questions），最后创建业务表（如 exams、papers、grading_results）
3. 初始数据：包含默认专业、班级、课程、题目、角色权限等基础数据，可直接用于系统测试
